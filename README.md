# BeatCfgMaker

> 音乐节拍配置工具 —— 用于手动标记音乐中的节拍时间点，并导出为游戏/应用可读的配置文件。

---

## 功能概述

BeatCfgMaker 是一个 WPF 桌面应用程序，允许用户：

1. **加载音频文件**，支持 mp3 / wav / flac / aac / ogg 格式
2. **实时播放音乐**，支持 0.5x / 1.0x / 1.5x / 2.0x 倍速
3. **按空格键打点**，在音乐播放过程中实时记录节拍时间戳
4. **管理多个节奏型**，每个节奏型可设置独立的循环监听次数和描述信息
5. **保存为配置文件**（`.beatcfg`），保存时自动对跨节奏型的相近时间点进行对齐格式化

---

## 使用流程

```
选择文件 → 填写节奏信息 → 开始配置 → 播放音乐 → 空格键打点 → 保存文件
```

### 详细步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 点击「选择文件」 | 选择一个本地音频文件 |
| 2 | 填写「循环监听次数」 | 该节奏型需要打点的次数，例如 2 或 4 |
| 3 | 填写「记录信息」 | 对该节奏型的描述，例如"主节奏"、"副歌" |
| 4 | 点击「开始配置」 | 进入录制模式，此时播放按钮变为可用 |
| 5 | 点击「播放」 | 开始播放音乐，**计时从此刻开始** |
| 6 | 按「空格键」 | 每次按下记录一个时间戳，达到循环次数后自动完成当前节奏型 |
| 7 | 点击「插入新节奏」 | 在当前节奏型完成后，插入下一个节奏型继续打点 |
| 8 | 点击「保存文件」 | 保存为 `.beatcfg` 配置文件 |

> **注意**：必须先点击「播放」再按空格键，计时以播放时刻为基准，确保时间戳与音乐同步。

## 输出文件格式

保存的 `.beatcfg` 文件为纯文本格式，结构如下：

```
beatRecords = {
    beat{key = 2, value = {{1200}{3400}}},
    beat{key = 4, value = {{1198}{3402}{5600}{7800}}}
}
```

| 字段 | 含义 |
|------|------|
| `key` | 该节奏型的循环监听次数 |
| `value` | 该节奏型所有时间戳的集合，单位为**毫秒**，每个时间点用 `{}` 包裹 |

---

## 时间对齐机制

保存时会自动对所有节奏型的时间戳进行**跨节奏型对齐**：

- **容差**：取所有节奏型中相邻时间点最短间隔的全局最小值
- **规则**：来自**不同节奏型**的时间点，若差值 < 容差，则统一替换为它们的平均值
- **保护**：同一节奏型内部的时间点**不会**被相互合并，只做跨节奏型对齐

这样可以修正因手动打点导致的微小误差，使不同节奏型中本应相同的时间节点保持一致。

